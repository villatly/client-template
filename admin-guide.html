1. Navigation Overview

The admin panel is accessible at /admin. The top navigation bar contains:

Nav ItemURLPurpose
Reservations/admin/bookingsView and manage all bookings
Availability/admin/availabilityBlock dates, set prices, manage iCal sync
Content/admin/contentAll guest-facing texts and media
Rooms/admin/roomsRoom types, images, descriptions, specs
Gallery/admin/galleryProperty-level photo gallery
Branding/admin/brandingColors, logo, favicon, layout, section colors
Sections/admin/sectionsShow/hide page sections
Booking/admin/bookingBooking mode, currency, WhatsApp, CTA
Settings/admin/settingsNotification email, timezone, deployment sync

The View Site → link (top right) opens your public website in a new tab.

2. Dashboard

The dashboard (/admin) shows a grid of quick-access cards, one per section. The Reservations card highlights in amber if there are bookings that need attention (pending review, payment failed, etc.).

The header displays the property name and type from Settings → Property Name and Content → Property Info → Property Type / Location.

3. Reservations

URL: /admin/bookings

Lists all bookings with their status, guest name, dates, room, and total price. Actions available per booking:

  • Accept / Decline — for Request to Book mode
  • Cancel — cancel a confirmed booking
  • View detail — guest info, payment status, admin notes

Booking statuses

StatusMeaning
Pending ReviewRequest submitted, awaiting your accept/decline
ConfirmedBooking active, dates blocked
Pending PaymentAwaiting guest payment (after acceptance)
Payment FailedStripe charge failed
CancelledCancelled by guest or admin
CompletedStay finished
Note: Bookings are not editable from this panel (dates, room, price). To manually create or adjust a booking, contact Villatly support.

4. Availability

URL: /admin/availability

Manage availability and pricing per room. Each room can have multiple physical units (e.g. Room 101, Room 102).

Fields per room

FieldDescription
Base Price per NightDefault nightly rate when no seasonal period applies
Min Stay (nights)Minimum number of nights required for a booking
Seasonal Price PeriodsCustom pricing for date ranges (e.g. High Season, Christmas). Each period has: label, from date, to date, price per night.
Blocked DatesDate ranges manually blocked (not available for booking). Shown in calendar view.
iCal SyncExternal calendars (Airbnb, Booking.com) imported as blocked dates. One feed per unit. See iCal section below.

iCal Sync (per unit)

FieldDescription
LabelName of the platform (e.g. "Airbnb", "Booking.com")
iCal URLThe webcal:// or https:// feed URL from the external platform
Last syncedTimestamp of last successful sync (read only)
Tip: iCal sync is one-way (import only). It pulls blocked dates from external platforms into Villatly's calendar. It does not push Villatly bookings back to those platforms.

5. Content

URL: /admin/content

All guest-facing text, images, and structured content is organized here in collapsible accordion sections. Each section has its own Save button — changes are not shared between sections.

5.1 Property Info

Basic metadata shown throughout the website (page header, hero eyebrow, location section, footer, emails).

FieldWhere it appearsNotes
Property NameWebsite header, hero eyebrow, footer, emailsKeep in sync with Settings → Property Name in Emails (stored separately)

Important: Property Name here controls what guests see on the website. The name in Settings controls what appears in email subject lines and the admin panel header. Update both when renaming your property.

10. Booking

URL: /admin/booking

Booking Mode

ModeHow it works
Instant BookGuest picks dates, pays via Stripe, and is confirmed immediately. Requires Stripe to be configured.
Request to BookGuest submits a request (no payment). You accept or decline. If accepted, the guest receives a Stripe payment link. Works without Stripe for receiving requests.

Online Payments (Stripe)

This panel shows whether Stripe is configured (active / not configured). To activate Stripe, add STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, and NEXT_PUBLIC_URL to your Vercel environment variables.

Booking Settings

FieldWhere it appearsNotes
CTA Button LabelBooking bar button, rooms CTA (default)e.g. "Check Availability", "Book Now", "Request a Stay"
CurrencyPrice displays throughout the site and in emailsISO 4217 code: USD, EUR, GBP, IDR, AUD, etc. Uppercase only.

Fallback Contact

FieldWhere it appearsNotes

Booking EmailFallback contact email, some email templatese.g. reservations@yourvilla.com
WhatsApp Number (master)Booking bar, floating button, Contact CTA section, all contact areas across the siteThis is the main WhatsApp field for the whole site. Include country code: +62 812 345 6789. This powers the floating WhatsApp button and the booking bar contact option.
Show floating WhatsApp buttonFloating circle button on all pages (bottom right corner)Checkbox. Visible only if a WhatsApp number is set. Uncheck to hide the button without removing the number.
Tip: The WhatsApp number here (Booking Settings) is the master for the whole site. The optional WhatsApp field in Content → Contact is only a fallback and is rarely needed.

11. Settings

URL: /admin/settings

Booking Notification Email

FieldNotes
Notification EmailWhere booking alerts are sent: new bookings, cancellations, payment failures, refund alerts. Can be any email address — it doesn't need to be your login email. Your admin login email is fixed in Vercel environment variables (ADMIN_EMAIL) and cannot be changed here.

Property Name in Emails

FieldNotes
Property NameAppears in email subjects ("Booking confirmed at Villa Amara") and in the admin panel header. Separate from Content → Property Info → Property Name (which controls the website). Update both when renaming your property.

Timezone

FieldNotes
Property TimezoneUsed to display check-in and check-out times correctly in emails and the admin panel. Select the timezone where your property is physically located.

Sync Content from Deployment

This action overwrites all live database content (texts, images, colors, pricing) with the JSON files bundled in the current Vercel deployment. Use this after pushing new template content to the server.

Destructive action: Clicking "Sync from deployment files" and confirming with the word SYNC will permanently replace all changes made through the admin panel with the deployment's seed files. This cannot be undone. Only use this if you intentionally want to reset to the last deployed content.

12. Field Overlap & Important Notes

Property Name — two fields, same value needed

LocationUsed forAction required
Content → Property Info → Property NameWebsite: header, hero, footer, guest-facing textUpdate both when renaming the property
Settings → Property Name in EmailsEmail subjects, admin dashboard title

WhatsApp — master vs fallback

LocationRoleWhat it powers
Booking → WhatsApp NumberMaster fieldBooking bar, floating button, Contact CTA, all contact sections
Content → Contact → WhatsApp NumberOptional fallbackOnly overrides the Contact section if the Booking field is empty. Rarely needed.

Rule: Always use the Booking Settings field. Leave the Contact field empty unless you specifically need a different number for the Contact section.

Hero CTA buttons — layout-dependent

LayoutHero CTA buttons shown?Hero interaction
Default (free)NoBooking bar (desktop) + scroll arrow
Editorial (premium)Yes — Primary + Secondary CTAButtons link to URL or open booking flow
Resort (premium)Yes — Primary + Secondary CTAButtons link to URL or open booking flow

The CTA Label/URL fields in Hero are stored but only visible to guests if you are on a premium layout. On the Default layout, those fields have no effect.

Room pricing — display vs actual

FieldLocationPurpose
Price FromRooms → Room DetailsStatic text label displayed on the room card. No calculations. e.g. "from $120/night".
Base Price per NightAvailability → RoomActual price used for booking calculations, calendar, and payment.
Seasonal Price PeriodsAvailability → RoomOverride the base price for specific date ranges.

Rule: Availability prices are what guests are charged. The "Price From" in Rooms is decoration only.
